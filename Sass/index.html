css: have to write .header 2 times .header { width: 100%; } .header ul { padding: 10px; } scss: only need to write header once .header { width: 100%; ul { padding: 10px; } } Benefits of Nesting The larger a site becomes, the more useful nesting becomes. It's especially useful when making changes to class names. If you wanted to change the class name of the header in the example below, you would need to make changes in 3 places ith normal CSS, but with Sass you only need to make one change. css .header { width: 100%; } .header ul { padding: 10px; } .header li { font-size: 15px; } .scss .header { width: 100%; ul { padding: 10px; li { font-size: 15px; } } }
